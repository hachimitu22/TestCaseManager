# TestCaseManager (tcm)

テストケースの作成・管理を支援する OSS ツールです。  
Node.js 上で動作し、CLI と Web サーバー機能を提供します。

## 目的

- テストケース作成者が、テストケースを継続的に管理できるようにする
- テストケース成果物を Git で扱いやすいファイルとして管理できるようにする
- 作成過程（組み合わせ表・判断理由）を含めて、テスト設計の知識を残せるようにする

## 現時点の方針

- 最終的に npm からインストールして使えるようにする
- 初期段階はツールとして実装を進め、公開可能な品質になった時点で npm パッケージとして公開する
- OSS として公開する（MIT License）
- DDD を採用し、ユビキタス言語を明確化しながら開発する

## MVP（npm 公開時の最小要件）

- テストケースの**作成**ができる
- テストケースの**一覧**ができる
- テストケースの**保存**ができる

## 実行形態

- `tcm init <dir>` で作業フォルダを初期化する
- `tcm server -p <port>` でローカル Web サーバーを起動する
  - 作業フォルダで実行する
  - デフォルトポートは `3000`

### 想定コマンド

```bash
tcm init <dir>
tcm server -p <port>
```

## データ保存方針

- 出力されるテストケースファイルは**すべて XML 形式**とする
- 内部実装で DB を使用してもよいが、ファイル出力を可能にする
- ファイル管理を前提とし、Git で差分管理しやすい構造を目指す

## テストケース記述形式

テストケースごとに、以下の形式を選択可能とする。

- AAA 方式
- Given-When-Then 方式
- テキスト記述方式

## ドメイン用語（初版）

- `testcase`
  - テストの最小単位
  - 準備・操作・確認を記述する
- `testsuite`
  - `testcase` をまとめた集合
  - グループ / カテゴリーとして扱う

## XML 仕様の考え方（初版）

- ツール専用タグは定義する
- それ以外の表現は過度に制限しない
- 実運用で必要な拡張に対応できる構造を優先する
